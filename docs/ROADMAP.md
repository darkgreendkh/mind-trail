# Mind Trail Roadmap

最后更新：2026-06-24

## 产品方向

Mind Trail 的长期方向是成为一个轻量、本地优先的学习路径导航工具。

它不是复杂笔记系统，也不是完整知识图谱。核心价值始终是：

- 让学习主线保持清楚。
- 让支线探索可控。
- 让用户随时知道自己现在在哪、从哪里来、下一步往哪里走。

## v0.1.0 - MVP

目标：跑通本地学习图的核心闭环。

范围：

- Project Home
- 多项目本地管理
- 默认打开上次项目
- Workbench 三段式布局
- React Flow 画布
- 节点选中与编辑
- 继续主线
- 创建支线
- 三种节点状态
- 删除节点子树
- 自动整理布局
- 回到当前节点
- localStorage 自动保存
- 刷新后恢复
- Enter / Tab / Delete 快捷键

不包含：

- 撤销 / 重做
- 搜索
- 导出
- AI
- 云同步
- 标签
- 多人协作

## v0.2.x - 编辑体验增强

目标：让日常使用更顺手，但不改变产品本质。

候选功能：

- 撤销 / 重做
- 搜索节点
- 更完整的快捷键
- 节点复制
- 更好的删除确认
- 更好的自动布局冲突处理
- 画布小地图
- 空状态和错误状态优化

优先级建议：

1. 撤销 / 重做
2. 搜索节点
3. 快捷键扩展
4. 自动布局优化

## v0.3.x - 本地数据能力

目标：让本地项目更可靠、更容易迁移。

候选功能：

- 导出 JSON
- 导入 JSON
- 导出 Markdown
- 数据版本迁移
- 本地备份
- 项目复制

注意：

- 导出能力不应过早影响 v0.1.0 的数据模型。
- 数据迁移需要等至少一个真实版本稳定后再设计。

## v0.4.x - 视图和组织增强

目标：在不做复杂知识图谱的前提下，提升大图可读性。

候选功能：

- 折叠 / 展开支线
- 聚焦某个子树
- 节点搜索结果高亮
- 主线视图
- 支线列表
- 简单统计，例如节点数量、完成数量、放弃数量

约束：

- 不引入复杂标签系统。
- 不把产品变成通用知识库。
- 不让画布失去“主线向下、支线向右”的核心结构。

## Later - 可选长期方向

这些方向可以探索，但不属于早期重点：

- 云同步
- 账号系统
- AI 总结
- AI 辅助生成学习路径
- 多端同步
- 多人协作
- 复杂知识图谱
- 插件系统

进入这些方向前，必须先验证：

- 用户是否真的长期使用本地学习图。
- 主线/支线模型是否足够有效。
- 大图情况下的导航体验是否仍然清晰。

## 当前原则

每个版本都应优先保护以下体验：

- 主线清晰。
- 支线可控。
- 当前节点明显。
- 操作少但顺手。
- 本地数据可靠。

如果一个功能会让画布更乱、概念更多、保存更脆弱，它应该推迟。
