# Mind Trail 产品规格

版本：v0.1.0  
状态：已实现（核心闭环可用）  
最后更新：2026-06-25

## 1. 产品定位

Mind Trail 是一个本地 Web 学习路径可视化工具。

它解决的核心问题是：学习者在探索知识时，经常从主问题跳到细节问题，最后忘记自己原来学到哪里。Mind Trail 用“主线向下、支线向右”的方式，把学习路径可视化，让用户始终知道：

- 我从哪里来。
- 我现在在哪。
- 下一步是继续主线，还是展开支线。

一句话定位：

> 像 VS Code 的跳转历史一样，帮助学习者在知识探索中不迷路。

## 2. v0.1.0 产品原则

必须坚持：

- 极简
- 本地优先
- 可视化优先
- 主线清晰
- 支线可控

v0.1.0 暂不做：

- 登录
- 云同步
- 导出
- AI 总结
- 复杂知识图谱
- 多人协作
- 标签系统
- 复杂统计

第一版只解决一个闭环：

> 创建一个学习图，在图中创建主线和支线，编辑节点，保存到本地，重新打开后继续学习。

## 3. 信息架构

v0.1.0 采用两层结构：

- Project Home：本地学习图列表。
- Workbench：单个学习图的画布编辑工作台。

应用启动规则：

- 首次打开或没有项目时，进入 Project Home。
- 有上次打开的项目时，默认进入该项目的 Workbench。
- Workbench 顶部必须提供返回 Project Home 的入口。

## 4. Project Home

Project Home 用于管理本地学习图。

v0.1.0 支持：

- 新建项目
- 打开项目
- 删除项目

规则：

- 删除项目必须二次确认。
- 项目标题不在 Project Home 做行内重命名。
- 项目标题在 Workbench 中编辑。
- 不做文件夹、搜索、排序筛选、导入导出。

## 5. Workbench 布局

Workbench 采用三段式布局：

- 顶部工具栏
- 中间/左侧主画布
- 右侧节点 Inspector

顶部工具栏包含：

- 项目标题
- 返回项目列表
- 自动整理布局
- 回到当前节点

主画布要求：

- 节点清晰
- 连线清晰
- 主线向下
- 支线向右
- 当前节点高亮
- 支持节点拖拽
- 支持画布缩放
- 支持画布平移
- 支持背景网格

右侧 Inspector 在选中节点后显示：

- 节点标题
- 节点状态
- 简短笔记
- 继续主线
- 创建支线
- 删除节点

## 6. 核心概念

### 节点

每一个节点代表一个学习点。

节点字段：

- 标题
- 状态
- 简短笔记
- 画布位置

节点状态只有三种：

- 进行中
- 已完成
- 已放弃

新建节点默认状态为进行中。

### 主线

主线表示学习的正常推进方向。

规则：

- 从当前节点向下扩展。
- 用主线边连接。
- 视觉上使用较粗实线。

示例：

```text
学 Agent Loop
↓
学 Prompt 部分
↓
学 Memory 部分
```

### 支线

支线表示从当前节点发散出去探索细节。

规则：

- 从当前节点向右扩展。
- 用支线边连接。
- 视觉上使用比主线更细的线。

示例：

```text
学 Prompt 部分 → Prompt 包含哪些 → Prompt Prefix 是什么
```

## 7. 核心交互

选中节点后，右侧 Inspector 提供这些动作：

- 继续主线
- 创建支线
- 修改状态
- 删除节点

交互规则：

- 点击节点即选中节点，并更新当前节点。
- 双击节点可直接行内编辑标题。
- 创建新节点后，新节点自动成为当前节点。
- 编辑标题和笔记时自动保存。
- 修改状态时自动保存。
- 拖拽节点位置后自动保存。
- 删除节点采用“删除子树”规则。

## 8. 删除规则

删除节点时，删除当前节点以及它派生出的所有后代节点。

后代定义：

- 从当前节点出发，沿主线边或支线边可以到达的所有节点。

删除时同时删除：

- 当前节点
- 所有后代节点
- 与这些节点相关的所有连线

删除后当前节点选择规则：

- 优先选择被删除节点的父节点。
- 如果没有可用父节点，则选择剩余节点中的一个节点。
- 如果项目被删空，则自动创建一个新的初始节点，保证项目至少有一个节点。

## 9. 自动布局规则

v0.1.0 的布局规则保持简单固定。

创建主线节点：

```text
x = 当前节点 x
y = 当前节点 y + 180
```

创建支线节点：

```text
x = 当前节点 x + 260
y = 当前节点 y + 140 * 当前节点已有支线数量
```

自动整理布局：

- 对全图重新计算位置。
- 保持主线向下。
- 保持支线向右。
- 会覆盖用户手动拖拽后的节点位置。

## 10. 节点视觉设计

节点使用圆角矩形。

建议尺寸：

- 宽度：180 - 220px
- 高度：80 - 120px
- 圆角：12px

状态视觉：

- 进行中：蓝色边框。
- 已完成：绿色边框，浅绿色背景。
- 已放弃：灰色边框，降低视觉权重。

当前节点视觉：

- 更粗边框。
- 轻微阴影。
- 右上角显示“当前”标记。

连线视觉：

- 主线：较粗实线，从节点底部边框中心引出，连到下方节点顶部。
- 支线：较细实线，从节点右侧边框中心引出，连到右侧节点左侧。
- 两条线都带箭头，指向子节点。
- 两者必须一眼可区分，但不要使用过度装饰。

## 11. 快捷键

v0.1.0 支持三个基础快捷键：

| 快捷键 | 行为 |
| --- | --- |
| Enter | 继续主线 |
| Tab | 创建支线 |
| Delete / Backspace | 删除当前节点 |

规则：

- 只有在已选中节点时快捷键才生效。
- 当用户正在输入标题或笔记时，快捷键不应抢占文本编辑行为。

## 12. 数据结构

localStorage 使用一个统一存储对象。

建议 key：

```text
mind-trail:v0.1.0
```

TypeScript 类型：

```ts
type NodeStatus = "in_progress" | "completed" | "abandoned";
type EdgeType = "main" | "branch";

type MindTrailStorage = {
  version: "0.1.0";
  activeProjectId: string | null;
  projects: MindTrailProject[];
};

type MindTrailProject = {
  id: string;
  title: string;
  currentNodeId: string;
  nodes: TrailNode[];
  edges: TrailEdge[];
  createdAt: number;
  updatedAt: number;
};

type TrailNode = {
  id: string;
  title: string;
  status: NodeStatus;
  note: string;
  x: number;
  y: number;
};

type TrailEdge = {
  id: string;
  from: string;
  to: string;
  type: EdgeType;
};
```

新项目默认数据：

- 项目标题：新的学习图
- 一个初始节点
- 初始节点标题：新的学习节点
- 初始节点状态：进行中
- 初始节点位置：`x = 100`，`y = 100`
- `currentNodeId` 指向初始节点

## 13. 推荐技术栈

v0.1.0 使用：

- Vite
- React
- TypeScript
- React Flow
- Zustand
- Tailwind CSS
- localStorage

职责划分：

- React Flow：画布、节点、连线、拖拽、缩放、平移。
- Zustand：项目、节点、边、当前节点、当前项目状态管理。
- localStorage：本地持久化。
- Tailwind CSS：基础样式。

## 15. 验收标准

v0.1.0 完成时必须满足：

- 首次打开应用时，如果没有项目，显示 Project Home。
- 用户可以创建一个新项目并进入 Workbench。
- Workbench 中默认有一个初始节点。
- 点击节点后，右侧 Inspector 显示节点信息。
- 用户可以编辑节点标题和笔记。
- 用户可以把节点状态改为进行中、已完成、已放弃。
- 点击“继续主线”后，在当前节点下方生成新节点和主线边。
- 点击“创建支线”后，在当前节点右侧生成新节点和支线边。
- 多条支线按固定间距向下排列。
- `Enter`、`Tab`、`Delete` 可以触发对应节点操作。
- 删除节点时会删除它的后代节点，不留下孤儿节点或悬空连线。
- 自动整理布局后，主线仍向下，支线仍向右。
- 用户可以返回 Project Home。
- 用户可以删除项目，删除前有确认。
- 刷新页面后，项目、节点、连线、状态、笔记、位置不会丢失。
- 再次打开应用时，默认进入上次打开的项目。

## 16. 非目标

v0.1.0 不追求复杂功能数量。

这版成功的标准不是“像完整知识管理工具”，而是：

> 打开快、路径清楚、创建顺手、刷新不丢、主线和支线一眼能看懂。
