# Changelog

## [0.1.0] - 2026-06-25

第一个可日常使用的版本，跑通了本地学习图的核心闭环。

### Added

- Project Home：多个本地学习图的新建、打开、删除（删除二次确认），默认打开上次项目。
- Workbench 三段式布局：顶部工具栏、React Flow 主画布、右侧节点 Inspector。
- 核心节点模型：标题、状态（进行中 / 已完成 / 已放弃）、笔记、画布位置。
- 主线（向下、较粗）与支线（向右、较细）两种连接，均带箭头。
- 节点操作：继续主线、创建支线、修改状态、删除节点（删除子树）。
- 双击节点行内编辑标题。
- 自动整理布局：全图重排为「主线向下、支线向右、父节点在分支上居中」。
- 工具栏「回到当前节点」聚焦当前节点。
- 画布缩放、平移、节点拖拽、背景网格，当前节点高亮。
- 撤销 / 重做：覆盖画布节点操作（删除、继续主线、创建支线、改状态、拖拽、自动布局、改标题/笔记），误删可一键恢复；会话级历史，连续文字编辑合并为一步。
- 快捷键：Enter 继续主线、Tab 创建支线、Delete / Backspace 删除、Ctrl+Z 撤销、Ctrl+Shift+Z / Ctrl+Y 重做。
- localStorage 同步持久化，刷新后完整恢复。
- 文档：`README.md`、`docs/PRODUCT_SPEC.md`、`docs/ROADMAP.md`、`docs/CHANGELOG.md`、`docs/architecture.md`、`docs/specs/` 实现计划。

### Changed

- 全新视觉风格：暖白纸感背景 + 鼠尾草绿强调色，正文换用霞鹜文楷（LXGW WenKai Screen）字体。
- 节点状态改用柔和色块整片填充区分（进行中浅绿 / 已完成深绿 / 已放弃暖灰并降透明度），状态前新增圆点标识；当前节点改为白底 + 绿色描边 + 外发光 + 右上角「当前」标签。
- 连线配色更新：主线为鼠尾草绿粗线、支线为暖灰细线；画布改为暖白纸感网点背景。
- 顶栏、右侧 Inspector、项目列表统一暖白纸感配色；Inspector 三个操作按钮采用三级深浅梯度。
- 顶栏项目标题输入框宽度自适应文字长度；节点状态圆点与标题那行字垂直居中；加高笔记输入区。

### Fixed

- 支线连线从节点右侧边框引出（而非底部），主线从底部引出，两者均加箭头。
- 自动整理布局：中段主线节点的支线现在与其父节点同高（此前会错误地停留在根节点那一行）。
- Backspace 删除走 store 真正删除节点（禁用 React Flow 内置删除，避免节点「复活」）。
- 点击节点偶尔需要点第二次才聚焦：鼠标按下时约 1px 的轻微抖动会落入 React Flow 的死区（点击被抑制、又不足以触发拖拽），导致当前节点不更新。现已将 `nodeClickDistance` 与 `nodeDragThreshold` 对齐（4px），轻微抖动仍算点击、节点不会被误移。
- 双击节点行内编辑标题时中文（输入法）输入异常，分两个原因修复：① 输入法组合中按 Enter 选词被 `onKeyDown` 当成"提交并关闭"——现已在 `e.nativeEvent.isComposing` 为真时让位给输入法；② 输入"shide"会变成"sshshishi'dshi'de"一长串：节点标题经 Canvas effect 同步进 React Flow，比按键滞后一拍，把这个旧值写回受控输入框会在组合输入中累积重复字符——现改为用本地 `draft` state 驱动输入框、失焦/回车时再提交到 store，输入框与所输内容始终同步。
